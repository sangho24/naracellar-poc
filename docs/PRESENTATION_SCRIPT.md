# 나라셀라 디지털 전환 PoC 발표 스크립트

> 삼일회계법인 AX Node | 발표자용 멘트 가이드
> 총 예상 시간: 15~20분

---

## 0. 오프닝 (1분)

> 화면: Overview 탭 — KPI 카드 4개

"안녕하세요. 삼일회계법인 AX Node입니다.

오늘 보여드릴 것은 단순한 기술 데모가 아닙니다.
나라셀라가 보유한 **1,955개 SKU, 570개 거래처, 10개 이상의 채널**,
이 방대한 데이터가 **의사결정을 어떻게 바꿀 수 있는지**를 직접 체감하실 수 있도록 준비했습니다.

지금 화면에 보시는 숫자들이 현재 나라셀라의 상태입니다.
재고보유일수 **362일**, 매출채권 연체율 **19.4%**, 환리스크 노출 **73억**.
이 숫자들이 어떻게 만들어졌는지, 그리고 어떻게 바꿀 수 있는지,
오늘 순서대로 보여드리겠습니다."

---

## 1. Overview — As-Is 진단 (2분)

> 화면: Overview 탭 전체

"먼저 현재 상황입니다.

상단 KPI를 보시면, 재고보유일수가 362일입니다.
2022년 215일이었던 것이 불과 3년 만에 147일이 늘었습니다.
창고에 1년치 재고가 쌓여 있다는 뜻입니다.

아래 차트를 보시면, 강남 도매 채널 매출이 3월에 급격히 꺾입니다.
전월 대비 **20% 하락**입니다.
원인은 거래처 폐업과 소비 패턴 변화인데, 지금은 이걸 알아채는 것 자체가 늦습니다.

그리고 오른쪽 Alert 리스트를 보시면 이미 시스템이 이상을 감지하고 있습니다.
클릭 한 번으로 원인 분석이 시작됩니다."

---

## 2. Root Cause 분석 — AI Agent (3분)

> 화면: Root Cause 분석 탭 → "Root Cause 분석 시작" 버튼 클릭

"Alert를 클릭하면 Root Cause 분석 탭으로 넘어옵니다.
여기서 **'Root Cause 분석 시작'** 버튼을 눌러보겠습니다.

*(버튼 클릭 후 Agent 실행 대기)*

지금 화면 왼쪽에서 AI Agent가 단계적으로 실행되고 있습니다.
이상 감지 → 데이터 수집 → 원인 분석 → 액션 추천,
이 4단계가 자동으로 진행됩니다.

오른쪽 그래프를 보시면, 강남 도매 지역의 거래처-SKU-브랜드 관계가 시각화됩니다.
빨간 점선으로 표시된 노드들이 폐업한 거래처입니다.
이 거래처들과 연결된 SKU들이 지금 재고로 쌓이고 있는 것입니다.

*(분석 완료 후)*

분석이 완료됐습니다. 원인 트리를 보시면:
- 1단계: 강남 도매 매출 20% 하락
- 2단계: 거래처 3곳 폐업, 프리미엄 수요 급감
- 3단계: MZ세대 위스키 이탈, 편의점 채널 구조적 성장

그리고 바로 아래에 **구체적인 액션 3개**가 추천됩니다.
실행 버튼을 누르면 시뮬레이션 결과까지 나옵니다.

*이것이 '감 기반 의사결정'에서 '데이터 기반 의사결정'으로의 전환입니다.*"

---

## 3. 와인 매칭 추천 — To-Be 핵심 데모 (5분)

> 화면: 와인 매칭 추천 탭

"이제 제가 오늘 가장 강조하고 싶은 부분입니다.

지금까지는 '문제'를 보여드렸다면,
지금부터는 **'해법'**을 보여드립니다.

*(탭 클릭, 3컬럼 화면 표시)*

왼쪽에 거래처 목록이 있습니다.
지금 선택된 곳은 **리스토란테 벨라**, 강남 신사동의 이탈리안 레스토랑입니다.
객단가 8만 5천원, 월 45병 구매하는 거래처입니다.

오른쪽 중앙을 보시면 AI가 추천한 와인 리스트가 있습니다.
잘 보시면 상위에 **주황색 'Long Tail' 배지**가 붙은 와인들이 있습니다.

이 와인, **도메인 드 라 코트 피노 누아**를 클릭해보겠습니다.
현재 재고 체류 **180일**입니다. 창고에 반 년째 있는 와인입니다.
그런데 이 와인의 매칭 스코어가 **87점**입니다.

왜냐하면, 이탈리안 레스토랑의 고객들은 섬세하고 산도 있는 레드를 선호하고,
이 와인의 가격대(소매가 8만 9천원)가 이 거래처의 객단가에 딱 맞기 때문입니다.

맞는 고객을 만나지 못했을 뿐인 겁니다.

*(거래처를 GS25 강남역점으로 변경)*

이번엔 거래처를 **GS25 강남역점**으로 바꿔보겠습니다.
추천 리스트가 완전히 바뀌었습니다.
객단가 1만 2천원, 편의점에 맞는 저가 와인 중심으로 재편됩니다.

1,955개 SKU가 하나하나 자기 자리를 찾아가는 것입니다.

상단 배너를 보시면 이 추천들이 실행될 경우
Long Tail 소진율이 **13.2%에서 개선**될 것으로 예상됩니다.

*'이 와인들이 나쁜 와인이 아닙니다. 맞는 고객을 만나지 못했을 뿐입니다.'*
*이것이 AI Agent가 하는 일입니다.*"

## 4. What-If 시뮬레이터 — 재무 효과 정량화 (5분)

> 화면: What-If 시뮬레이터 탭

"마지막으로, 이 모든 전환이 재무적으로 얼마나 의미 있는지 보여드리겠습니다.

*(탭 클릭)*

지금 모든 슬라이더가 0입니다. 현재 상태입니다.
영업손실 **-2.3억**.

슬라이더를 하나씩 움직여 보겠습니다.

---

**먼저 '보수적' 시나리오부터.**

*(보수적 버튼 클릭)*

수요예측 정확도 10%, 환헤지 30%, 연체 조기식별 40%, AI 추천 10%.
가장 보수적으로 잡은 수치입니다.

KPI 변화를 보시면:
- 재고보유일수 362일 → 즉각 줄어들기 시작합니다
- 매출채권 연체율도 떨어지고
- Long Tail 소진율도 올라갑니다

아래 워터폴 차트가 항목별 기여도를 보여줍니다.
이자비용 절감, 환리스크 절감, 대손 절감, IT비용 절감, Long Tail 매출 증가.

---

**이번엔 'Base Case'.**

*(기본 버튼 클릭)*

*(화면의 영업이익 숫자가 플러스로 전환되면 강조)*

보시다시피 영업이익이 **플러스로 전환**됩니다.
현재 영업손실 -2.3억이 흑자로 돌아섭니다.

---

수요예측 슬라이더를 직접 올려보겠습니다.

*(슬라이더 천천히 드래그)*

수요예측 정확도가 1% 올라갈 때마다 재고보유일수가 줄어드는 것 보이시나요?
재고가 줄면 차입금이 줄고, 이자비용이 줄고, 현금이 돌아옵니다.
데이터 하나가 연쇄적으로 재무를 개선합니다.

---

핵심 메시지를 말씀드리면:

**비용 절감 효과만으로도 현재 영업손실의 흑자 전환이 가능합니다.**
Long Tail 활성화는 여기에 추가적인 매출 성장입니다.

클라우드 인프라와 자동화가 현금흐름을 개선하고,
AI가 매출을 성장시키는 구조입니다."

> 화면: Overview 탭으로 복귀

"오늘 보여드린 것을 정리하면:

1. **감지** — 이상 신호를 자동으로 포착합니다
2. **분석** — AI Agent가 근본 원인을 단계적으로 추적합니다
3. **추천** — 1,955개 SKU를 맞는 거래처에 연결합니다
4. **정량화** — 전환 효과를 억원 단위로 시뮬레이션합니다

나라셀라가 보유한 1,955개 와인은 나쁜 와인이 아닙니다.
570개 거래처는 충분히 많은 고객입니다.
이 조합을 영업사원의 머리로 풀 수는 없습니다.
**이건 AI Agent의 영역입니다.**

감사합니다. 질문 받겠습니다."

## 예상 질문 & 답변

**Q. 실제 데이터와 연동이 되나요?**
> "현재 PoC는 목데이터 기반입니다. ERP(SAP/더존) API 연동, POS 데이터 연결, 실제 거래처 데이터 적재는 Phase 1 구현 범위에 포함됩니다. 데이터 파이프라인 구축 후 동일한 UI에서 실시간으로 작동합니다."

**Q. AI 추천이 틀릴 수도 있지 않나요?**
> "맞습니다. 그래서 모든 추천에 신뢰도 점수와 근거가 함께 표시됩니다. '실행' 버튼은 영업사원이 판단해서 누르는 구조입니다. AI는 '결정'이 아니라 '옵션'을 제공합니다."

**Q. 구축 비용과 기간은 어느 정도인가요?**
> "제안서 로드맵 기준, Phase 1(인프라 + 자동화) 3개월, Phase 2(분석 + 대시보드) 3개월, Phase 3(AI Agent 고도화) 6개월입니다. 상세 견적은 별도 협의 드리겠습니다."

**Q. 기존 시스템과 충돌하지 않나요?**
> "클라우드 기반으로 기존 온프레미스 시스템과 병행 운영이 가능합니다. API 레이어로 연결하기 때문에 현행 업무 중단 없이 단계적으로 전환합니다."
