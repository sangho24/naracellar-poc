경로: `/output/poc_sku_analysis.pdf`

- **성공 시**: 실데이터로 가격대×품종×생산지 클러스터링
- **실패 시**: 1,955 SKU / 161 브랜드 / 상위 4 = 93% 매출 정보 기반으로 plausible synthetic distribution 생성. 페이지 하단에 *"외부 추정 모델 기반"* 명시.

분석 내용:
- 5~8개 군집으로 클러스터링
- **위험 SKU 50개** 후보: 가격대/품종이 최근 한국 시장 트렌드 (위스키 이탈, 무알코올, 편의점 와인 강세)와 mismatch + 회전 낮을 가능성
- **기회 SKU 50개** 후보: entry-level / 편의점 채널 적합 / MZ 전환 가능 후보

시각화 (1 페이지에 다 들어가야 함):
- 클러스터 산점도 또는 heatmap
- Risk/Opportunity matrix
- 핵심 인사이트 3줄 (예: "프리미엄 이탈리아 X섹터에서 33% SKU가 12개월 내 deadstock 위험")
- 하단 disclaimer: *"공개 데이터 기반 외부 분석. 내부 POS/거래처 데이터 결합 시 정확도 대폭 향상"*

## 5. 매니저 미팅 talking script (Markdown)
경로: `/output/manager_meeting_script.md`

30~45분 미팅 가정.

- **오프닝 (30초)**: 매니저에게 시간 요청하는 카톡/메시지 1개 + 미팅 시작 멘트 1개
- **핵심 설명 (10~15분)**: deck 3장 페이지별 핵심 메시지를 *말하는 형태*로 정리
- **POC 공유 (5분)**: 1-pager 설명 + 의도된 메시지 ("이거 정확도가 핵심이 아니라, 외부 데이터만으로 이만큼이 보인다는 메시지입니다")
- **자문 요청 (10분)**: 매니저에게 묻고 싶은 5개 open question. 카테고리:
  1. Sponsorship 전략 (상무 vs 다른 파트너)
  2. Deck reframing (cash flow 각도가 맞는지)
  3. POC 정확도와 방향성
  4. 제휴 owner 접근 방법
  5. 다음 액션 우선순위
- **클로징**: 다음 step 합의 후보 옵션 2~3개

# 제약사항

- **Over-polish 금지**. 모든 결론을 단정적으로 쓰지 말 것. 일부 항목은 의도적으로 *"이 부분은 매니저님 의견 듣고 싶음"* 등 open question으로 둘 것.
- **사용자에게 확인 질문 금지**. 정보 부족하면 가정을 명시하고 자율로 진행. 사용자가 매니저 미팅 전에 자율로 마무리해야 함.
- 모든 숫자/주장은 출처 또는 가정 명시.
- 시각화는 단순/명료 우선. 화려한 디자인 X.
- 한국어 비즈니스 톤 (격식체와 구어체 사이의 중간 — 컨설팅 매니저급에게 자연스러운 정도).
