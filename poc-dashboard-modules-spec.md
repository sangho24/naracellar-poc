# POC 대시보드 추가 모듈 구현 요청서

> **프로젝트:** 나라셀라 디지털 전환 제안서 — POC 대시보드
> **기술 스택:** React (웹 기반)
> **목적:** 발표 시 To-Be 비전을 체감시키기 위한 인터랙티브 데모 2종

---

## 배경 컨텍스트

### 나라셀라 개요
- 국내 1호 와인 유통 상장사 (2023년 코스닥 상장)
- 161개 브랜드, 1,955개 SKU, 570개 거래처, 10+ 채널 운영
- 미국(케이머스 외) 42%, 프랑스(부샤 외) 18%, 칠레(몬테스 외) 16% — 상위 3개국이 매출 76%
- **핵심 문제:** 소수 히트 브랜드에 매출 편중 → 나머지 SKU는 재고로 적체 → 재무 악화

### 주요 재무 지표 (2025년 3분기 기준)
| 지표 | 값 | 비고 |
|------|-----|------|
| 재고보유일수 | 362일 | 2022년 215일 대비 +147일 |
| 매출채권 연체율 | 19.4% | 연체 금액 25.4억 |
| 환리스크 노출 | 외화부채 73억 | 5% 변동 시 3.6억 영향 |
| Long Tail 소진율 | 13.2% | 비인기 SKU 판매 비중 |
| 현금 | 44억 | vs 부채 316억 |
| 재고평가충당금 | 2.86억 | 시장가 < 원가인 재고 |
| 영업이익 | -2.3억 (누적) | 3Q 단독은 +2억 흑자전환 |

### 기존 대시보드 구성
현재 POC 대시보드에는 다음이 구현되어 있음:
- **상단 KPI 카드 4종:** 재고보유일수(362일), 매출채권 연체율(19.4%), 환리스크 노출(73억), Long Tail 소진율(13.2%)
- **Knowledge Graph 기반 시각화:** 브랜드-거래처-채널 관계 네트워크
- **Root Cause 분석 기능:** 재무 문제의 원인 추적

→ **As-Is 진단은 강하지만, To-Be(전환 후 모습)를 체감시키는 데모가 없음.**
→ 아래 2개 모듈을 추가하여 "전환하면 이렇게 작동하고, 재무적으로 이만큼 좋아진다"를 보여줘야 함.

## 모듈 1: 거래처 × 와인 매칭 추천 데모

### 목적
제안서 Phase 3 "AI 개인화 추천 Agent"의 프로토타입. 발표 시 "이런 UX로 영업사원이 쓰게 됩니다"를 실제로 보여주는 것이 목표.

### 핵심 메시지
> "이 와인들이 나쁜 와인이 아닙니다. 맞는 고객을 만나지 못했을 뿐입니다."

1,955개 SKU 중 Long Tail에 해당하는 와인도 맞는 거래처에 연결되면 팔릴 수 있다는 것을 시각적으로 증명.

실제 나라셀라 포트폴리오 기반. 히트 브랜드와 Long Tail을 섞어야 함:

**와인 구성 비율:**
- 히트 브랜드 (재고체류 < 60일): 10~15개
- 중간 (60~120일): 15~20개
- Long Tail (120일 초과): 25~40개 ← 이게 많아야 데모의 의미가 살아남

#### 매칭 로직 (룰 기반)
ML 모델이 아니어도 됨. 아래 룰을 스코어링으로 구현:

```
매칭 스코어 = (가격대 적합도 × 0.3) + (카테고리 적합도 × 0.25) 
            + (업종 궁합 × 0.2) + (재고 소진 긴급도 × 0.15) 
            + (마진 기여도 × 0.1)
```

- **가격대 적합도:** 와인 소매가가 거래처의 priceRange 안에 들어가면 높은 점수
- **카테고리 적합도:** 거래처의 preferredCategory와 와인의 country/grape/type 매칭
- **업종 궁합:** 이탈리안 레스토랑 ↔ 이탈리아 와인, 와인바 ↔ 다양한 품종, 편의점 ↔ 저가 와인 등
- **재고 소진 긴급도:** stockDays가 높을수록 가산점 (Long Tail SKU를 우선 추천)
- **마진 기여도:** margin이 높을수록 가산점

#### 추천 이유 텍스트 자동 생성
각 추천에 1~2줄의 이유를 표시. 예시:
- "이 거래처의 객단가(8.5만원)에 적합한 가격대이며, 이탈리안 메뉴와 피에몬테 레드의 궁합이 높습니다."
- "현재 재고 체류 180일 초과 SKU이나, 유사 업종(와인바)에서 월 8병 판매 실적이 있습니다."
- "이 상권(연남동)의 MZ 고객 선호도가 높은 내추럴 와인 카테고리입니다."

### 인터랙션 요구사항
1. 거래처 선택 시 추천 리스트가 실시간 갱신
2. 추천 와인 클릭 시 상세 정보(와인 프로필 + 매칭 근거) 확장
3. **핵심:** 추천 리스트에서 Long Tail SKU가 상위에 노출되는 경우를 반드시 포함할 것. 히트 브랜드만 추천되면 데모의 의미가 없음
4. 상단 또는 사이드에 "이 추천으로 Long Tail 소진율 13.2% → X% 개선 예상" 같은 요약 지표 표시

### 디자인 톤
- 기존 대시보드와 일관된 스타일
- 다크 모드 기반 (기존 대시보드가 다크 모드라면)
- 프로페셔널하고 깔끔한 UI. 삼일회계법인 제안서답게 신뢰감 있는 톤
- 컬러 강조: Long Tail SKU에는 시각적 구분 (예: 태그 또는 뱃지)

### 시뮬레이션 로직

#### 슬라이더 1: 수요예측 정확도 개선 (0% ~ 40%)
```
현재 재고보유일수 = 362일
개선 비율 = slider_value (%)

# 예측 정확도 개선 → 과잉 발주 감소 → 재고 감소
new_inventory_days = 362 - (362 - 215) × (slider_value / 40)
# 215일 = 2022년(최적 상태) 기준, 40% 개선 시 2022년 수준 회복

# 재고 감소 금액 (현재 재고 약 400억 추정 기준)
inventory_reduction = (362 - new_inventory_days) / 362 × 400억

# 이자비용 절감 (차입이자율 약 5% 가정)
interest_saving = inventory_reduction × 0.05
```

#### 슬라이더 2: 환율 헤지 자동화 비율 (0% ~ 100%)
```
외화부채 = 73억
환율변동_영향 = 3.6억 (5% 변동 시)

# 헤지 비율만큼 노출 감소
risk_reduction = 3.6억 × (slider_value / 100) × 0.5
# × 0.5는 "헤지가 모든 손실을 막지는 않는다"는 보수적 가정
```

#### 슬라이더 3: 연체 조기식별률 (0% ~ 100%)
```
현재_연체금액 = 25.4억
현재_대손충당금_연증가 = 2억

# 조기식별 → 사전 조치 → 대손 감소
bad_debt_saving = 2억 × (slider_value / 100) × 0.7
# × 0.7은 "식별해도 100% 방지는 못한다"는 보수적 가정

# 연체율 변화
new_overdue_rate = 19.4% × (1 - slider_value / 100 × 0.6)
```

#### 슬라이더 4: AI 추천 Long Tail 활성화 (0% ~ 50%)
```
현재_long_tail_rate = 13.2%
연매출 = 800억 (근사치)

# AI 추천 → Long Tail 판매 비중 증가
new_long_tail_rate = 13.2% + slider_value
additional_revenue = 연매출 × (slider_value / 100) × 0.3
# × 0.3은 "추천한다고 다 팔리지는 않는다"는 보수적 가정
```

#### 종합 절감 효과
```
total_saving = interest_saving + risk_reduction + bad_debt_saving + IT비용절감(고정 ~30%)
total_with_revenue = total_saving + additional_revenue

current_operating_loss = -2.3억
projected_operating = current_operating_loss + total_saving
```

### 인터랙션 요구사항
1. 슬라이더 조작 시 모든 KPI가 실시간(또는 부드러운 애니메이션으로) 갱신
2. Before → After 수치가 나란히 보여야 함 (변화량도 표시)
3. 종합 재무 영향 섹션에 워터폴 차트 또는 막대 차트로 절감 항목별 기여도 시각화
4. **프리셋 시나리오 3종** 제공 (버튼 클릭으로 슬라이더 자동 설정):
   - `Conservative (보수적)`: 수요예측 10%, 헤지 30%, 식별 40%, LT 10%
   - `Base Case (기본)`: 수요예측 20%, 헤지 50%, 식별 60%, LT 20%
   - `Optimistic (낙관적)`: 수요예측 35%, 헤지 80%, 식별 80%, LT 40%
5. 현재 영업손실(-2.3억) 대비 흑자/적자 전환 여부를 시각적으로 강조 (흑자 시 그린, 적자 시 레드)

### 디자인 톤
- 모듈 1과 동일한 스타일 유지
- 슬라이더는 직관적이고 터치 친화적으로 (발표 시 마우스/터치로 조작)
- 숫자 변화에 카운트업/다운 애니메이션 적용
- 워터폴 차트는 recharts 또는 d3로 구현

### 발표 시 사용 시나리오

**모듈 1 시연 순서:**
1. 거래처 "리스토란테 벨라" (이탈리안) 선택
2. 추천 리스트에 히트 브랜드가 아닌 이탈리아 Long Tail 와인이 상위에 노출되는 것을 보여줌
3. "이 와인은 현재 재고 체류 180일이지만, 이 업종에서는 매칭 스코어 94입니다"
4. 거래처를 "GS25 강남점"으로 전환 → 완전히 다른 추천 리스트 (저가 와인 중심)
5. "이것이 1,955개 SKU가 제자리를 찾는 방법입니다"

**모듈 2 시연 순서:**
1. 모든 슬라이더 0% 상태에서 시작 (현재 상태)
2. "Conservative" 프리셋 클릭 → "보수적으로 잡아도 연 X억 절감"
3. "Base Case" 프리셋 클릭 → "영업손실 -2.3억이 흑자로 전환됩니다"
4. 수요예측 슬라이더를 직접 올리면서 → "수요예측 정확도 1%가 올라갈 때마다 재고보유일수가 이만큼 줄어듭니다"
5. "비용 절감만으로도 흑자 전환이 가능하고, Long Tail 활성화는 추가 성장입니다"
