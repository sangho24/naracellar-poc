## 2. 분석 알고리즘

### 2-1. 클러스터링 접근
**선택한 방식**: 머신러닝 K-means 대신 **해석 가능한 룰 기반 세분화**.

**이유**:
- 가격·재고회전 같은 연속형 변수 부재 → K-means 입력 부적합
- 매니저·client에게 *"왜 이 SKU가 위험으로 분류됐는가"* 즉답 가능해야 함
- POC 단계에서는 설명력 > 통계적 우아함

**Risk Score 산식** (직관적 가중합, 0~100 정규화 대상):
```
risk = country_oversupply(0~15)         # 국가 SKU/매출 gap (정의: 2-3)
     + (type==Red) * 15                  # 한국 시장 레드 51%로 축소 중
     + (no_awards) * 20                  # 시장 검증 신호 부재
     + (max_score < 90) * 10             # 점수 낮음 (있어도)
     + (type in {Dessert, Fortified}) * 25  # 회전 느린 틈새
     + (country∈{FR,IT} & Red) * 10      # mid-premium 양극화 직격
```

**Opportunity Score 산식**:
```
opp  = (type in {White, Sparkling}) * 30  # 성장 카테고리
     + (type==Rosé) * 15                  # 대안 카테고리
     + has_awards * 15                    # 시장 검증
     + (max_score ≥ 90) * 20              # 결정적 검증
     + growing_varietal * 20              # Riesling/Sauv.Blanc/Chard/Pinot Noir/Champagne
     + (country∈{Chile, NZ, Argentina, S.Africa}) * 10  # entry-level + 편의점 적합
```

### 2-2. Long Tail Gap 정의
국가별:
```
gap = (해당 국가 SKU 비중 %) - (해당 국가 매출 비중 %)
```
- gap > 0: SKU는 많은데 매출은 적음 = **Long Tail 미소진 위험**
- gap < 0: SKU 대비 매출이 큼 = **under-curated, 더 큐레이션 가능**

### 2-3. 핵심 정량 결과

| 국가 | SKU 수 | SKU 비중 | 매출 비중 | gap |
|------|--------|---------|----------|-----|
| **France** | 280 | 37.8% | 23.0% | **+14.8%p (가장 큰 위험)** |
| USA | 185 | 25.0% | 36.0% | -11.0%p (under-curated) |
| Italy | 114 | 15.4% | 7.0% | **+8.4%p (위험)** |
| Chile | 41 | 5.5% | 13.0% | -7.5%p (under-curated) |
| Spain | 32 | 4.3% | (기타 21% 内) | +4.3%p |
| Australia | 25 | 3.4% | (기타) | +3.4%p |
| Germany | 17 | 2.3% | (기타) | +2.3%p |
| Portugal | 15 | 2.0% | (기타) | +2.0%p |

**카테고리 분포 (740 SKU)**:
- 레드 50.4% (한국 시장 51.5% — 거의 일치)
- 화이트 31.6% (시장 33.7% — 거의 일치)
- 스파클링 12.2% (시장 9% — 오히려 앞섬)
- 로제 1.8%, 디저트·주정강화 2.6%

> 카테고리 mismatch는 **유의미한 미스매치 없음**. 진짜 문제는 **국가별 mismatch** (특히 프랑스 +15%p).

## 4. 한계 — 정직하게

### 4-1. 외부 데이터로는 불가능한 영역
| 분석 항목 | 외부 데이터 가능? | 내부 데이터 결합 시 |
|----------|------------------|---------------------|
| **재고 회전율** | ❌ 불가 (가격·재고일수 비공개) | ✅ SKU별 평균 체류일 측정 |
| **거래처별 판매 이력** | ❌ 불가 | ✅ 570 거래처 × 1,955 SKU 매칭 행렬 |
| **채널별 (도매/리테일/온라인) 분리** | ❌ 불가 | ✅ 영업1/영업2/리테일 채널별 회전율 |
| **계절성·시즌 효과** | ❌ 불가 | ✅ 시계열 패턴 학습 |
| **가격대별 회전** | ❌ 불가 (가격 비공개) | ✅ 가격대 × 회전율 매트릭스 |
| **신용도·연체 패턴** | ❌ 불가 | ✅ 거래처 신용스코링 |

### 4-2. 본 PoC가 *증명하려는 메시지*
> **정확도가 메시지가 아닙니다.**
> 외부 공개 데이터만으로 이 정도까지 보입니다 → 내부 POS·거래처 데이터 결합 시 정확도는 비약적으로 올라갑니다.
> 이게 본 PoC의 의도된 메시지이며, "위험 SKU 50개 명단"의 개별 정확도는 부차적.

### 4-3. 알려진 편향
1. **2026 1Q 매출 비중**은 1개 분기 데이터 — 연간 평균과 다를 수 있음
2. **awards 채움률 40%** — 어워드 없다고 다 실패한 SKU는 아님 (신규·소규모 와이너리 포함)
3. **국가 매출 기타 21%** — Spain/Australia/Germany/Portugal 비중은 추정만 가능
4. **vintage 정보 0%** — 동일 이름이지만 다른 빈티지인 SKU 다수 (실제로는 더 많은 active SKU 가능성)

## 5. 내부 데이터 결합 시 개선 시나리오

본 PoC가 외부 데이터로 *방향성*을 잡았다면, 내부 데이터 결합 1주 안에 다음이 가능:

1. **위험 SKU 50개 → 위험 SKU 200개 전수** 회전율 기반 재선별
2. **거래처별 매칭 행렬** — "이 위험 SKU를 받아줄 가능성이 가장 높은 거래처 TOP 10" 산출
3. **편의점 채널 적합도 점수** — 기회 SKU 50개의 가격대×브랜드 인지도 매핑
4. **얼로케이션 자동 추천** — 어떤 SKU를 어떤 채널에 얼마나 배분할지 What-If
5. **재고 100억 감소 시뮬레이션** — Phase 2의 핵심 효과 정량 입증

> 본 PoC는 **이 결합 시점부터 진짜 가치가 나오기 시작**합니다. 외부 데이터 분석은 *입구*입니다.
